# 엔드포인트 보안 자가점검·자동조치 툴킷

> Windows · macOS 단말의 보안 설정을 한 번에 점검하고, 위험이 낮은 항목은 자동으로 조치한 뒤, 결과를 Google Sheet로 모으는 도구입니다. IT 전담 인력이 없는 소규모 조직에서 비전문가도 더블클릭만으로 실행할 수 있도록 만들었습니다.

Windows는 30개, macOS는 16개 항목을 점검합니다. 점검 항목과 판정 기준은 「주요정보통신기반시설 기술적 취약점 분석·평가」의 PC/Windows 점검 영역, 국가정보원 PC 보안 점검 항목, ISMS-P 단말 통제, 그리고 Microsoft·Apple 보안 기준선에 근거합니다.

## 개요

회사 보안 기준에 맞춰 단말의 설정 상태를 점검하고, 안전하게 바꿀 수 있는 항목은 자동으로 조치합니다. 점검과 조치 결과는 화면과 텍스트·JSON 파일로 남고, 설정에 따라 Google Sheet로도 모입니다.

- **대상**: Windows 10/11 (Home 포함), macOS 13(Ventura) 이상 · Apple Silicon/Intel
- **실행 권한**: Windows는 관리자, macOS는 설정 변경 시 로그인 암호 1회
- **소요 시간**: 1~2분
- **사용 환경**: 별도 에이전트·서버 없이 단말에서 직접 실행하는 단발성 점검 도구

> 로그인 자체를 막지 않으며, 개인 파일(문서·사진·메시지)은 읽지도 보내지도 않습니다. 보안 설정 상태와 조치 내역만 수집합니다.

## 주요 특징

- **점검 + 자동 조치 통합**: 점검만 하고 끝내지 않고, 위험이 낮은 항목은 같은 실행에서 바로 조치합니다.
- **비전문가 대상 설계**: 항목마다 무엇을 보는지·왜 중요한지를 풀어 쓰고, 직접 바꿔야 하는 항목은 설정 경로까지 안내합니다.
- **저위험 항목만 자동 조치**: 사용·데이터에 영향을 줄 수 있는 항목(디스크 암호화, UAC, 패스워드 만료 주기 등)은 자동 조치에서 빼고 '권고'로만 표시합니다.
- **되돌리기 제공**: 조치 전 상태를 백업하고, macOS는 원복 스크립트를 생성합니다.
- **결과 집계**: 여러 대의 결과(JSON)를 엑셀 한 장으로 합치거나, Google Sheet로 자동 전송합니다.

## 동작 방식

한 번 실행하면 아래 순서가 자동으로 이어집니다.

```
0. 단말 정보 수집(자산)
        ↓
1~5. 보안 점검(영역별 항목 탐지)
        ↓
   점검(탐지) 요약 — 조치 전 상태(양호 / 취약 / 권고 / 정보)
        ↓
   저위험 항목 자동 조치 (--check-only 지정 시 생략)
        ↓
   조치 결과 출력 + 백업/원복 파일 생성
        ↓
   조치 후 재점검 — 조치한 항목을 다시 읽어 '조치 전 → 조치 후' 비교, 최종 판정 갱신
        ↓
   최종 요약 + 텍스트·JSON 저장 → (설정 시) Google Sheet 전송
```

조치 후 자동으로 다시 점검하므로, 결과를 확인하려고 두 번 실행할 필요가 없습니다. 바뀐 항목은 "조치 전 → 조치 후"로 함께 표시되고, 최종 요약·JSON·시트에는 조치 후 상태가 반영됩니다. 다만 패스워드 정책 등 일부 설정은 사용자의 다음 로그인/재부팅부터 완전히 적용됩니다.

**판정 4단계**

| 판정 | 의미 |
|------|------|
| 양호 | 기준을 충족함 |
| 취약 | 기준 미달, 조치 필요 |
| 권고 | 보안상 바람직하나 사용·데이터 영향이 있어 자동 조치하지 않음 |
| 정보 | 환경에 따라 달라지는 참고 항목(대개 정상) |

**조치 3단계**

| 표시 | 의미 |
|------|------|
| 자동 조치 | 이 도구가 같은 실행에서 직접 처리 |
| 직접 조치 | 설정 화면에서 사용자가 직접 변경(경로 안내 제공) |
| 담당자 문의 | 정책·판단이 필요해 정보보호 담당자가 처리 |

## 점검 항목

영역 구분(계정·서비스·패치·로그·보안 관리)은 「주요정보통신기반시설 기술적 취약점 분석·평가」의 점검 체계를 따릅니다. `PC-` 항목은 표준 PC 보안 점검 항목, `ADD-`·`MAC-` 항목은 단말 환경에 맞춰 보강한 항목입니다. 판정 기준값(예: 잠금 10회, 화면보호기 10분)은 일반 기준이며 사내 정책에 맞게 조정할 수 있습니다.

## 자동 조치 범위

자동 조치는 **되돌리기 쉽고 사용에 영향이 없는 저위험 설정**으로 한정합니다.

**Windows 자동 조치(14개)**: 패스워드 최소 길이·복잡성·최소 사용기간, 계정 잠금 정책, Guest 비활성화, 불필요/위험 서비스 중지, SMBv1 비활성화, 방화벽 활성화, 감사 정책, 익명 열거 제한, 미로그온 종료 해제, 이동식 미디어 자동실행 차단, 자동 로그온 해제, 화면 보호기(대기·암호).

**macOS 자동 조치(8개)**: 화면 잠금 암호, 자동 로그인 비활성화, 게스트 비활성화, Gatekeeper 활성화, 자동 보안 업데이트, 방화벽(+스텔스), 원격 로그인(SSH) 비활성화, 원격 관리(ARD) 비활성화.

### 자동 조치에서 제외한 항목과 근거

| 항목 | 제외 사유 |
|------|-----------|
| 디스크 암호화 (BitLocker / FileVault) | **복구 키를 먼저 안전하게 보관하지 않고 켜면 데이터 잠김 위험.** 키 escrow가 전제되어야 함 |
| UAC 활성화 (Windows) | 권한 요청 창 동작이 바뀌어 사용에 영향 |
| 패스워드 최대 사용기간 | 최신 가이드(NIST SP 800-63B 등)는 주기적 강제 만료를 권장하지 않음 |
| 마지막 로그온 사용자명 숨김 | 로그인마다 사용자명 직접 입력이 필요해져 마찰 발생 |
| 로그온 경고 배너 | 로그인마다 확인 클릭이 추가됨 |
| SIP (macOS) | 비활성 시 위험하나 활성화는 복구 모드 작업이 필요 |
| 파일 공유(SMB) | 복합기 스캔→폴더 등 업무에 필요할 수 있어 일괄 차단하지 않음 |

> **디스크 암호화는 의도적으로 자동 활성화하지 않습니다.** 복구 키를 먼저 확보(MDM, Google Workspace 기기관리, Microsoft 계정/Entra ID 등으로 escrow)한 뒤 적용해야 합니다. 이 도구는 키를 수집하지 않습니다.

### 환경에 따른 자동 판정 보정

거짓 '취약'을 줄이기 위해, 일부 항목은 단말 환경을 읽어 판정을 보정합니다.

- **PC-06 패치(Windows)**: Windows Update 검색 API는 Defender 시그니처와 단계적 배포(아직 이 PC 차례가 아닌) 항목까지 '미설치'로 보고합니다. 이 도구는 **분류(Definition Updates)와 자동 선택 여부(AutoSelectOnWebSites) 같은 속성 기준**으로 그런 항목을 미적용 집계에서 제외해, 사용자가 Windows Update 창에서 실제로 설치할 수 있는 패치만 셉니다. (KB 이름을 박지 않으므로 다음에 다른 항목이 와도 자동으로 적용됩니다. 제외된 항목은 보고서 근거에 함께 표시합니다.)
- **PC-10 방화벽(Windows)**: 백신 등 타사 보안 제품이 Windows 방화벽을 대신 관리하는 환경에서는, Windows 방화벽이 꺼져 있어도 거짓 '취약'으로 보지 않고 '양호/정보'로 보고하며 Windows 방화벽 강제 켜기를 생략합니다(켜도 제품이 다시 끄는 충돌 방지). 판단은 Windows 보안 센터에 등록된 방화벽/백신 제품으로 합니다.
- **macOS Sequoia(15)**: Gatekeeper(`spctl`)나 화면 잠금처럼 명령으로 더는 바꿀 수 없는 항목은, 잘못된 값을 쓰지 않고 시스템 설정 경로 안내로 대체합니다.

## 근거 자료

각 점검 항목을 공식 점검 기준과 항목 단위로 매핑했습니다. Windows 항목은 대부분 KISA 「주요정보통신기반시설 기술적 취약점 분석·평가 상세가이드」의 **Windows 서버 점검 항목(W-01~W-64)** 과 직접 대응하고, PC 환경 고유 항목은 행정안전부·국가정보원 **내PC지키미** 점검 항목에 대응합니다. macOS는 국내 공식 단일 점검표가 없어 **CIS Apple macOS Benchmark** 를 기준으로 했습니다.

**일치 표기**: `정확` = 공식 항목과 1:1 대응 · `유사` = 상위/인접 항목에 포함되거나 범위가 일부 다름 · `기준선` = 공식 점검표에는 없고 벤더 보안 기준선에 근거.

### 출처

- **KISA 「주요정보통신기반시설 기술적 취약점 분석·평가 방법 상세가이드」** — Windows 서버 점검 항목(W-01~W-64) 및 PC 점검 영역. 영역 구분(계정·서비스·패치·로그·보안 관리)이 본 도구와 일치 (KISA 보호나라&KrCERT 자료실)
- **행정안전부·국가정보원 「내PC지키미」 점검 항목** — 보안업데이트·패스워드 안전성·화면보호기·공용폴더·USB 자동실행·네트워크 설정 등 PC 환경 점검 항목
- **CIS Apple macOS Benchmark** — macOS 보안 설정 기준(국내 공식 단일 점검표 부재로 채택)
- **ISMS-P 인증기준** — 단말/PC 보안 통제의 상위 근거
- **Microsoft / Apple 보안 기준선** — 공식 점검 코드가 없는 일부 항목(UAC 등)의 근거

> **검증 범위 안내**: Windows 항목의 W-코드 대응은 KISA 가이드의 공개된 항목 목록과 대조해 작성했습니다. 다만 W-코드 번호는 가이드 발간 연도에 따라 일부 바뀔 수 있으니, 인증·심사용으로 인용할 때는 적용 대상 연도의 가이드 원문을 기준으로 최종 확인하시기 바랍니다. macOS는 CIS 벤치마크의 통제 '주제' 수준으로 대응시켰으며, 정확한 통제 번호는 macOS 버전별로 다릅니다.

## 사용법

### 1. 내려받기 / 구조

저장소를 내려받으면(또는 릴리스 zip을 풀면) 아래 구조입니다.

```
endpoint-security-toolkit/
├── windows/        # Windows용: 보안점검조치_실행.bat, win_sec.ps1, sheet_config.txt
├── macos/          # macOS용:   Mac_보안점검조치.command, sheet_config.txt
├── google-sheet/   # (선택) 결과 수집용 Apps Script + SETUP.md
└── tools/          # 결과 집계 스크립트 aggregate_to_excel.py
```

각 플랫폼 폴더 안 파일들은 **같은 폴더에 그대로** 두어야 합니다(스크립트가 같은 폴더의 `sheet_config.txt` 를 읽습니다). 배포할 때는 해당 플랫폼 폴더만 압축해 사용자에게 전달하면 됩니다.

### 2. (선택) Google Sheet 연동 설정

결과를 여러 대에서 한 시트로 모으려면, **본인 소유의** Google Sheet에 Apps Script(`google-sheet/google_sheet_endpoint.gs`)를 직접 배포한 뒤, 각 플랫폼 폴더의 **`sheet_config.txt`** 에 그 웹앱 URL을 넣습니다.

```
SHEET_URL=https://script.google.com/macros/s/AKfyc...(본인 URL).../exec
SHEET_TOKEN=your-secret-token
```

배포·설정 전체 절차는 **[google-sheet/SETUP.md](google-sheet/SETUP.md)** 를 참고하세요. 시트를 안 쓰면 비워두면 됩니다(점검·조치는 정상 동작).

> 이 저장소에는 어떤 수집 서버 주소도 들어 있지 않습니다. 결과에는 자산 정보가 포함되니 **반드시 본인 시트에 모으고 공유 범위를 제한**하세요.
>
> **`sheet_config.txt` 만 수정하세요. 스크립트 본체(`.ps1`/`.command`)는 편집하지 마세요** — 편집기로 저장하면 인코딩이 깨져 다른 PC에서 실행 오류가 날 수 있습니다([주의사항](#주의사항) 참고).

### 3. 실행

#### Windows

`windows/` 폴더의 `보안점검조치_실행.bat` 을 더블클릭합니다. 관리자 권한 요청(UAC)이 뜨면 [예]를 누릅니다. 점검 → 자동 조치 → 결과 출력이 한 번에 진행됩니다.

#### macOS

`macos/` 폴더의 `Mac_보안점검조치.command` 를 더블클릭합니다. 인터넷에서 받은 파일이라 "악성 코드를 확인할 수 없습니다" 경고가 한 번 뜹니다. 정상이며, 한 번만 허용하면 됩니다.

- **macOS 15(Sequoia) 이상**: 경고창 [취소] → 시스템 설정 → 개인정보 보호 및 보안 → 맨 아래 [그래도 열기] → 암호/Touch ID → [열기]
- **macOS 14(Sonoma) 이하**: 파일 우클릭(Control+클릭) → [열기] → 다시 [열기]
- **터미널 방식(빠름)**: 터미널에 `xattr -dr com.apple.quarantine ` 입력 후(끝에 한 칸) 폴더를 드래그하고 Enter

설정 변경 단계에서 로그인 암호(또는 Touch ID)를 한 번 물을 수 있습니다.

#### 실행 옵션

| 옵션 (Windows / macOS) | 설명 |
|------------------------|------|
| `-CheckOnly` / `--check-only` | 점검만 하고 자동 조치는 생략(테스트용) |
| `-Force` / `--force` | 담당자 이메일 질문 없이 완전 무인 실행 |
| `-OwnerEmail` / `--email` | 담당자(자산 식별) 이메일 지정 |
| `-SheetUrl` / `--sheet-url` | 설정 파일 대신 명령줄로 URL 전달(우선 적용) |

예시:

```powershell
# Windows — 점검만
powershell -ExecutionPolicy Bypass -File win_sec.ps1 -CheckOnly
```

```bash
# macOS — 점검만
bash Mac_보안점검조치.command --check-only
```

### 4. 결과 집계

여러 대의 JSON 결과를 엑셀 한 장으로 합칩니다.

```bash
python3 tools/aggregate_to_excel.py
# 같은 폴더의 WinSec_*.json / MacSec_*.json 을 모아 보안점검_통합_<날짜>.xlsx 생성
```

`요약`, `항목별`, `조치내역` 3개 시트로 정리됩니다. Google Sheet 연동을 켰다면 별도 집계 없이 시트의 `점검결과`·`상세` 탭에서 바로 확인할 수 있습니다.

## 산출물

실행하면 스크립트와 같은 폴더에 다음이 생성됩니다(쓰기가 막히면 바탕화면).

| 파일 | 내용 |
|------|------|
| `WinSec_<호스트>_<시각>.txt` / `MacSec_...txt` | 사람이 읽는 점검·조치 보고서 |
| `WinSec_<호스트>_<시각>.json` / `MacSec_...json` | 집계용 구조화 결과(데이터 스키마 `winsec/1`, 도구 버전 `toolVersion` 포함) |
| `WinSec_백업_<호스트>_<시각>` (Windows) | 조치 전 설정 백업 |
| `MacSec_원복_<호스트>_<시각>.command` (macOS) | 조치 전으로 되돌리는 스크립트 |

## 수집·미수집 데이터

- **수집**: 단말 식별 정보(호스트명·모델·시리얼·OS·IP/MAC 등 자산 정보), 보안 설정 점검 결과, 자동 조치 내역, 담당자 이메일
- **미수집**: 문서·사진·메시지 등 개인 파일 내용, 웹 검색·방문 기록, 입력 내용

수집 항목은 보안 설정 상태와 자산 식별에 한정됩니다.

## 되돌리기(원복)

- **Windows**: 조치 전 설정을 `WinSec_백업_...` 에 저장합니다.
- **macOS**: `MacSec_원복_...command` 를 실행하면 조치 전 상태로 되돌립니다.
- 자동 로그인이 꺼져 시작 시 암호 입력이 생기는 등 변화가 불편하면 위 파일로 복구하거나 담당자에게 문의하면 됩니다.

## 주의사항

- **스크립트 본체를 편집기로 열어 저장하지 마세요.** Windows `.ps1` 은 한글 처리를 위해 UTF-8 BOM으로 저장되어 있는데, 편집기 저장 시 BOM이 빠지면 한국어 코드페이지(CP949) 환경에서 한글이 깨져 파싱 오류가 납니다. 설정은 `sheet_config.txt`(영문이라 인코딩 무관)에만 넣으세요.
  - macOS `.command` 는 반대로 **BOM이 없어야** 합니다(셔뱅이 깨짐). `.ps1` 에만 BOM이 필요합니다.
- **관리자/암호 권한**이 없으면 일부 항목은 '정보'로 표시되고 조치가 적용되지 않습니다.
- 실제 적용 효과는 각 OS에서만 검증됩니다. 배포 전 **`--check-only` 로 점검만 먼저 돌려** 결과를 확인하길 권합니다.
- 배포 채널(Slack·Drive 등)을 거치면 macOS는 매번 Gatekeeper 경고가 뜹니다. 반복 운영 시에는 코드 서명·공증 또는 MDM 배포로 이 단계를 없앨 수 있습니다.

## 버전

현재 **v1.0** (첫 공식 릴리스). 도구 버전은 실행 화면·보고서 헤더, JSON의 `toolVersion`, 그리고 (연동 시) 시트의 '도구버전' 컬럼에 기록됩니다. Windows·macOS는 같은 버전 번호로 함께 배포합니다.

## 기여

이슈·PR 환영합니다. 점검 항목 추가나 기준값 변경은 근거(공식 점검표/벤더 기준선)를 함께 적어 주세요. 스크립트 본체를 수정할 때는 인코딩 규칙을 지켜 주세요 — `win_sec.ps1` 은 **UTF-8 BOM**, `*.command` 는 **BOM 없는 UTF-8** 이어야 합니다.

## 면책

이 도구는 **현재 상태 그대로(as-is)** 제공되며, 사용에 따른 책임은 사용자에게 있습니다. 단말 설정을 자동으로 변경하므로, **배포 전 반드시 `--check-only`(macOS) / `-CheckOnly`(Windows) 로 점검만 먼저 돌려** 결과를 확인하고, 대표 단말에서 시범 적용한 뒤 확대하시길 권합니다. 디스크 암호화처럼 데이터에 영향을 줄 수 있는 항목은 자동 조치에서 제외되어 있습니다.
